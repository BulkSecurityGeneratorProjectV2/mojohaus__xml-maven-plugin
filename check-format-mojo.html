xml:check-format

Full name:

org.codehaus.mojo:xml-maven-plugin:1.0.1:check-format

Description:

An XML indentation check over a set of files.

Attributes:

  • Requires a Maven project to be executed.
  • The goal is thread-safe and supports parallel builds.
  • Binds by default to the lifecycle phase: validate.

Optional Parameters

Name Type Since Description
catalogs String[] - An XML catalog file, or URL, which is being used to resolve entities.
encoding String - The encoding of files included in formatFileSets. Note that the encoding can be set also per FormatFileSet.
Default value is: ${project.build.sourceEncoding}.
User property is: xml.encoding.
failOnFormatViolation boolean - Tells the mojo what to do in case XML formatting violations are found. if true, all violations will be reported on the console as ERRORs and the build will fail. if false, all violations will be reported on the console as WARNs and the build will proceed further.
Default value is: true.
User property is: xml.failOnFormatViolation.
formatFileSets List - File patterns to include. The patterns are relative to the current project's baseDir.
indentSize int - The number of spaces expected for indentation. Note that indentSize can be configuread also per FormatFileSet.
Default value is: 2.
User property is: xml.indentSize.
skip boolean - Whether to skip execution.
Default value is: false.
User property is: xml.skip.
useDefaultFormatFileSet boolean - If set to true, the result of FormatFileSet.getDefault() will be appended to formatFileSets before the processing.
Default value is: true.
User property is: xml.useDefaultFormatFileSet.

Parameter Details

An XML catalog file, or URL, which is being used to resolve entities.

  • Required: No

The encoding of files included in formatFileSets. Note that the encoding can be set also per FormatFileSet.

  • Required: No
  • User Property: xml.encoding
  • Default: ${project.build.sourceEncoding}

Tells the mojo what to do in case XML formatting violations are found. if true, all violations will be reported on the console as ERRORs and the build will fail. if false, all violations will be reported on the console as WARNs and the build will proceed further.
  • Type: boolean
  • Required: No

  • Default: true

formatFileSets:

File patterns to include. The patterns are relative to the current project's baseDir.
  • Type: java.util.List
  • Required: No

indentSize:

The number of spaces expected for indentation. Note that indentSize can be configuread also per FormatFileSet.
  • Type: int
  • Required: No
  • User Property: xml.indentSize
  • Default: 2

skip:

Whether to skip execution.
  • Type: boolean
  • Required: No
  • User Property: xml.skip
  • Default: false

useDefaultFormatFileSet:

If set to true, the result of FormatFileSet.getDefault() will be appended to formatFileSets before the processing.
  • Type: boolean
  • Required: No
  • User Property: xml.useDefaultFormatFileSet
  • Default: true
