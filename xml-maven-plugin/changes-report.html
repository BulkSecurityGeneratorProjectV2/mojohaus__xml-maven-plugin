Release 1.0.1 – Not yet released

Type Changes By
Fixed an NPE when specifying the schema via public ID. Fixes MOJO-1648. Thanks to Paul Lucassen. jochen
Wrong property name in documentation (transformation.html) Fixes MOJO-1690. Thanks to Jeff Martin. jochen

Release 1.0 – 2011-Feb-14

Type Changes By
Disabled proxy password logging. Also reduced logging level of proxy info from "info" to "debug". Fixes MOJO-1593. Thanks to Mykola Nikishov. jochen
The ResourceManager is now used when loading stylesheets or resolving included files, thus enabling use of classpath resources. Fixes MOJO-735. Thanks to Mark Hobson. jochen
Documentation fix: Replace ReplaceRegExpFileMapper with RegExpFileMapper. Fixes MOJO-1512. Thanks to Ramón Torres Bátiz. jochen
Catalogs can now be specified as URL's, not only as files. Fixes MOJO-1471. Thanks to Andreas Eternach. jochen
The resolver does now attempt to use the namespace URI and not only the public ID. Fixes MOJO-1470. Thanks to Andreas Eternach. jochen
Add support for in-place transformation. Fixes MOJO-1447. Thanks to Luke W. Patterson. jochen
Added catalog file example to the docs. Fixes MOJO-1413. jochen

Release 1.0-beta-3 – 15-Sep-2009

Type Changes By
External DTD's are no longer downloaded when checking for well formedness. Fixes MOJO-1120. Thanks to Michael Brackx. jochen
Provide log output with log-level INFO for the processing of any input file. Fixes MOJO-1167. olamy
Added the possibility to set a transformers output properties. Fixes MOJO-741. jochen
If a transformation sets or validation sets base directory is missing, then a warning is logged now, and the build no longer fails. This allows to configure the plugin in a parent POM. Fixes MOJO-742. jochen
It is now configurable, whether the Resolver creates validating parsers. Fixes MOJO-740. jochen
A parameter for specifying the transformer implementation has been added. Fixes MOJO-777. Thanks to Aleksei Valikov. jochen
Added support for the resource: protocol. Fixes MOJO-735. jochen

Release 1.0-beta-2 – 11-Apr-2007

Type Changes By
Fixed a misleading error message. Fixes MOJO-608. Thanks to Jonathan Card. jochen
Add support for XSLT parameters when transforming. Thanks to Andrew Thornton. jochen
SAXSource is now used instead of StreamSource, in order to ensure that the resolver is used. Thanks to Andrew Thornton. jochen
XML schema validation wasn't turned on by the ValidateMojo. Thanks to Panda C. jochen
The plugin does now honor the Maven proxy settings. Fixes MOJO-378. jochen
Added support for file mappers when transforming files. Fixes MOJO-631. jochen
Added docs for using an XSLT 2.0 compliant processor. jochen
Added support for resolving resources through the classloader. Fixes MOJO-735. jochen

Release 1.0-beta-1 – 29-Nov-2006

Type Changes By
Add use of the default excludes. Also add the configuration parameter skipDefaultExcludes for turning them off. Fixes MOJO-446. Thanks to Anagnostopoulos Kostis. jochen
Absolute paths have been unsupported. Fixes MOJO-448. Thanks to Anagnostopoulos Kostis. jochen
The plugin does now check, whether the generated files are uptodate. Fixes MOJO-453. Thanks to Anagnostopoulos Kostis. jochen

Type Changes By
Initial release jochen
